在大规模分布式深度学习推理服务中，KV cache 的内存管理是性能与稳定性的核心问题。When the model generates tokens autoregressively, each layer produces a key and value tensor that must be cached to avoid recomputation. The size of this cache grows linearly with the sequence length, batch size, and the number of layers. 对于具有数百亿参数的 MoE 模型，如 Qwen3.5-397B-A17B-FP8，每个 token 的 KV cache 占用约 80KB（FP8 量化后），因此长上下文（如 32K 或 64K tokens）会占用数 GB 显存。To handle this efficiently, paged attention splits the cache into fixed-size pages and uses an indirection table to map logical positions to physical pages. This avoids fragmentation and allows for efficient prefix sharing via radix tree caching. 当多个请求共享相同的 system prompt 时，radix cache 可以避免重复计算 prefix 的 KV，仅对新 token 做 prefill 计算。这就是为什么客户日志中会出现 '#new-token: 232, #cached-token: 53852' 这样的批次。
