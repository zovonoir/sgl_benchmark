

【段落 #1】
【产品背景】淘宝天猫AIGC创意生成平台旨在面向千万级商家提供端到端的智能化广告素材生成服务。
该平台需要在保证创意质量、商品要素一致性、品牌调性、合规性的前提下，实现秒级返回，支撑
日均数亿次的素材请求。底层依赖的核心能力包括：多模态大模型推理、品牌风格知识库检索、人工
审核与反馈闭环、A/B 实验调度、流量分桶与精细化定价。本系统的关键技术挑战集中在三个方面：
第一，超长上下文下推理性能与显存占用的平衡；第二，混合架构模型（注意力 + 状态空间）在
prefix cache 命中场景下的正确性；第三，FP8 量化在长尾输入分布下的数值稳定性。

【System Prompt - Brand Guideline Excerpt】You are an expert advertising creative
assistant specializing in e-commerce product copywriting. Your responses MUST adhere
to the following brand voice principles when generating ad copy, headlines, product
descriptions, banner slogans, and short-video scripts: (1) Avoid superlative claims
such as "best", "first", "lowest price" unless backed by verifiable data; (2) Comply
with PRC Advertising Law, GB/T standards, and the Tmall content moderation policy
for restricted categories including health products, cosmetics, financial services,
education, real estate, and infant formula; (3) Maintain a tone that is warm, trustworthy,
and conversational; do not use aggressive sales language or false urgency such as
"only 1 left", "ends today" unless quantitatively verified by the merchant's stock API;
(4) When asked to produce variations, generate at least 5 distinct copy alternatives
with different emotional appeals (rational, emotional, social-proof, FOMO, aspiration).

【商品要素清单 - Schema】每条商品输入包含以下结构化字段：商品ID(item_id)、类目路径
(cat_path)、品牌(brand)、关键卖点(selling_points: List[str])、价格区间(price_range)、
目标人群(target_audience)、风格标签(style_tags: List[str])、合规要素(compliance_flags)、
历史CTR(history_ctr)、库存状态(stock_status)、当前活动(active_campaigns)。生成的创意素
材必须显式覆盖所有 selling_points，且不得包含 compliance_flags 之外的医疗或绝对化表述。

【Tool Calling Schema】You have access to the following tools and MUST use them when
appropriate (qwen3_coder parser): {"name": "fetch_product_history", "description":
"Retrieves the 30-day creative-rendering history for a given item_id, useful for
deduplication and style continuity.", "parameters": {"item_id": "string"}}, {"name":
"check_compliance", "description": "Runs the candidate ad copy through Tmall's
real-time compliance checker; returns a list of flagged sentences and suggested
rewrites.", "parameters": {"copy": "string", "category": "string"}}, {"name":
"render_banner", "description": "Submits a banner-rendering job to the GPU farm and
returns a render_id that can be polled.", "parameters": {"template_id": "string",
"copy_lines": "List[str]", "color_palette": "List[str]"}}. Tool calls must be emitted
in the structured xml-like format expected by the qwen3_coder parser. Never invent
tools that are not in this schema; never call tools recursively beyond depth 3.

【历史对话与偏好记忆 - Long Context】在用户ID为 U102837465 的长期偏好画像中，记录了
以下要素：用户偏好简约风格的视觉、对节日营销有较强响应、过去30天最佳CTR来自亲子家居类
目（CTR 4.7%）、用户主动屏蔽过过度营销话术，文案中禁止出现"必买"、"全网最低"、"不容
错过"等词汇。用户的店铺历史素材中，颜色偏好为低饱和度的莫兰迪色系，构图偏好为留白与
对称，字体偏好为思源宋体 / Source Han Serif。我们已基于该用户的过往10000条创意素材
做了离线挖掘，得出的 top-3 文案模板为：(a) 故事场景式 + 价值主张收尾；(b) 数据型对
比 + 限时活动结尾；(c) 情感共鸣 + 用户证言收尾。该用户对模板 (a) 的响应率显著最高。

【Inference Backend Constraints】The inference runtime is SGLang with the ATOM
plugin (`SGLANG_EXTERNAL_MODEL_PACKAGE=atom.plugin.sglang.models`) on a 4-way TP
configuration over AMD MI308X GPUs. The attention backend is `aiter`, with FP8
prefill attention disabled (`SGLANG_AITER_FP8_PREFILL_ATTN=0`) and FP8 decode
attention enabled. Chunked prefill is set to 2048 tokens; max running requests is
256. The model is Qwen3.5-MoE with hybrid full-attention + Mamba/Gated-DeltaNet
layers, FP8 weights and activations (per-1x128 quantization for the MoE expert
GEMMs), and a context window of 128K. Radix cache is enabled (no
`--disable-radix-cache`), which means hybrid Mamba state recomputation is invoked
on every prefix-cache hit.

【User Memory】关键事实备忘：用户姓名李明，工作单位上海某互联网公司，团队规模约30
人，主要负责本地生活类目的广告创意自动化。技术栈使用 Spark 3.5 + Iceberg 1.5 + 
Flink 1.18 + Doris 2.1，日均处理日志量约 10TB，存储格式 Parquet。团队目前在
推进 LLM 在创意生成、商品理解、风险审核三个方向的落地。用户对推理框架、量化技术、
KV cache 优化、CUDAGraph、Mamba 状态缓存、prefix cache 都有较深入的工程经验。

【多语言示例 / Multilingual examples】1. 「初秋上新 · 一件式连衣裙温柔登场」/
"Early autumn new arrivals — one-piece dresses make a gentle debut" / 「秋の新作・
ワンピースが優しく登場」/ "Свежие осенние новинки — нежные платья дебютируют". 2.
「家庭清洁 · 让家更亮一点」/ "Home cleaning, a brighter touch for your home" /
「お家のお掃除、もっと明るいおうちに」/ "Уборка дома — пусть ваш дом сияет ярче".
3. 「学习陪伴 · 让孩子爱上每一次思考」/ "Learning companions: help kids fall in
love with thinking" / 「学習のおとも・お子さまが考えることを好きになる」/ "Учебные
помощники: пусть дети полюбят размышлять". 重复以上多语言示例可以显著扩充上下文，同时
保留语义的多样性，便于触发 BPE 分词器的不同子词路径，对 prefix cache 与 mamba state
recomputation 的覆盖更加充分。
